## 1. تسجيل الدخول

### الوصول إلى التطبيق

- **وضع التطوير**: `http://localhost:5173`
- **وضع الإنتاج**: `http://localhost:3002`
- **تطبيق سطح المكتب**: تشغيل التطبيق من قائمة ابدأ

### تسجيل الدخول

1. أدخل عنوان البريد الإلكتروني في حقل **البريد الإلكتروني**
2. أدخل كلمة المرور في حقل **كلمة المرور**
3. انقر على **تسجيل الدخول**

**حساب المسؤول الافتراضي:**

| الحقل          | القيمة               |
|---------------|----------------------|
| البريد الإلكتروني | `admin@khazane.dz` |
| كلمة المرور    | `admin123`           |

> ⚠️ قم بتغيير كلمة المرور فور تسجيل الدخول الأول في بيئة الإنتاج.

بعد تسجيل الدخول بنجاح، يتم توجيهك إلى **لوحة التحكم**.

## 2. لوحة التحكم

لوحة التحكم هي الصفحة الرئيسية للتطبيق. تعرض ملخصاً شاملاً للنشاط.

### بطاقات الإحصائيات

أربعة مؤشرات في الوقت الحقيقي معروضة في أعلى الصفحة:

| المؤشر                | الوصف                                   |
|----------------------|------------------------------------------|
| **المنتجات النشطة**   | العدد الإجمالي للمنتجات في النظام       |
| **الحركات الأخيرة**   | عدد آخر الحركات                         |
| **التنبيهات النشطة**  | تنبيهات عتبة المخزون المفعّلة           |
| **إجمالي العمليات**   | الحجم الكلي للعمليات                    |

### الإجراءات السريعة

أربعة أزرار تسمح بإنشاء حركة مخزون بسرعة:

- **إدخال** (أخضر) — استلام البضائع
- **إخراج** (أحمر) — شحن / استهلاك
- **نقل** (أزرق) — نقل بين المواقع
- **تعديل** (رمادي) — تصحيح الجرد

النقر على زر يفتح مباشرة نموذج الحركة معبأ مسبقاً بالنوع المقابل.

### آخر الحركات

يعرض جدول آخر 10 حركات مخزون مع:
- مرجع الحركة
- النوع (شارة ملونة: أخضر/أحمر/أزرق/أصفر)
- المنتج المعني
- الكمية
- التاريخ والوقت

## 3. إدارة المنتجات

متاحة عبر القائمة الجانبية **المنتجات**.

### عرض المنتجات

- تعرض القائمة جميع المنتجات مقسمة إلى صفحات (20 لكل صفحة)
- استخدم **شريط البحث** في الأعلى للتصفية حسب الاسم أو SKU أو الرمز الشريطي
- تنقل بين الصفحات بأزرار التنقل في الأسفل

### إنشاء منتج

1. انقر على زر **+ منتج جديد**
2. املأ النموذج:

| الحقل             | الوصف                          | مطلوب |
|------------------|--------------------------------|:-----:|
| **الاسم**         | اسم المنتج                    | ✅    |
| **SKU**           | رمز وحدة التخزين الفريد       | ✅    |
| **الفئة**         | فئة المنتج                    | ❌    |
| **وحدة القياس**   | وحدة، كغ، غ، لتر، مل، م، م²، م³، كرتون، منصة نقالة، حصة | ✅    |
| **الرمز الشريطي** | الرمز الشريطي EAN/UPC         | ❌    |

3. انقر على **إنشاء**
4. يظهر المنتج في القائمة

### استيراد المنتجات (Excel/CSV)

1. انقر على زر **استيراد** (أيقونة التحميل)
2. اختر ملف Excel (`.xlsx`) أو CSV (`.csv`)
3. يتم الاستيراد تلقائياً
4. رسالة تأكيد تعرض عدد المنتجات المنشأة والمحدّثة
5. في حال وجود أخطاء، يُعرض عدد الأسطر الخاطئة

### تصدير المنتجات (Excel)

1. انقر على زر **تصدير** (أيقونة التنزيل)
2. يتم تنزيل ملف `produits-khazane.xlsx` تلقائياً
3. يحتوي الملف على جميع المنتجات بمعلوماتها الكاملة

## 4. حركات المخزون

متاحة عبر القائمة الجانبية **الحركات**.

### أنواع الحركات

| النوع        | الأيقونة | اللون  | الاستخدام                                |
|-------------|---------|--------|------------------------------------------|
| **إدخال**    | ↓       | أخضر   | استلام، شراء، إنتاج                     |
| **إخراج**    | ↑       | أحمر   | شحن، استهلاك، بيع                       |
| **نقل**      | ↔       | أزرق   | نقل من موقع إلى آخر                     |
| **تعديل**    | ⚙       | أصفر   | تصحيح الجرد، تلف، انتهاء صلاحية        |

### إنشاء حركة

1. انقر على أحد الأزرار **إدخال** أو **إخراج** أو **نقل** في أعلى الصفحة
2. املأ النموذج:

| الحقل              | الوصف                              | مطلوب      |
|-------------------|-------------------------------------|:----------:|
| **المنتج**         | اختر المنتج المعني                 | ✅         |
| **الكمية**         | عدد الوحدات                        | ✅         |
| **موقع المصدر**    | مصدر المخزون (إخراج/نقل)           | حسب النوع |
| **موقع الوجهة**    | وجهة المخزون (إدخال/نقل)           | حسب النوع |
| **المورد**         | المورد (إذا كان النوع إدخال)     | ❌         |
| **العميل**         | العميل (إذا كان النوع إخراج)     | ❌         |
| **ملاحظات**        | تعليق حر                           | ❌         |

3. انقر على **تأكيد**
4. يتم تحديث المخزون تلقائياً
5. تظهر الحركة في القائمة وعلى لوحة التحكم

### عرض السجل

- تعرض القائمة جميع الحركات مقسمة إلى صفحات (20 لكل صفحة)
- كل حركة تعرض: المرجع، النوع (شارة ملونة)، المنتج، المورد، العميل، الكمية، المشغل، التاريخ

### بون الإخراج (قابل للطباعة)

يمكن إنشاء بون إخراج قابل للطباعة من صفحة الحركات:

1. انقر على زر **بون الإخراج** في أعلى الصفحة
2. يظهر مستند منسق يحتوي على:
   - رقم بون مُولَّد تلقائياً (`BS-YYYYMMDD-HHMM`)
   - جدول آخر حركات **الإخراج** (المرجع، المنتج، الكمية)
   - حساب تلقائي **لإجمالي الكميات**
   - مناطق التوقيعات (سُلِّم من / استُلِم من / المسؤول)
3. انقر على **طباعة** لفتح نافذة الطباعة في المتصفح

> يتكيف بون الإخراج تلقائياً مع اللغة النشطة (RTL للعربية).

## 5. التنبيهات

| الحالة         | اللون  | الأيقونة | الوصف                                  |
|---------------|--------|---------|----------------------------------------|

| **مُقرّ به**    | أصفر   | 🕐      | تم الأخذ بعين الاعتبار، قيد المعالجة   |

- **المنتج** المعني

## 6. التخزين (مواقع / مناطق / أماكن)

متاح عبر القائمة الجانبية **التخزين**.

### التسلسل الهرمي للتخزين

يستخدم التطبيق تنظيماً من ثلاثة مستويات:

```
الموقع (مستودع / متجر)
  └── المنطقة (ممر / قطاع / رصيف)
       └── المكان (رف / حامل / منصة نقالة)
```

### عرض المواقع

تعرض الصفحة قائمة مواقع التخزين مع:
- **الاسم**
- **الرمز** الفريد (مثال: `ENT-01`)
- **النوع** (Warehouse، Store، إلخ)
- **العنوان**

### البيانات المُعدّة مسبقاً (seed)

| المستوى   | الرمز          | الاسم             | التفاصيل                      |
|----------|----------------|-------------------|-------------------------------|
| موقع     | ENT-01         | المستودع الرئيسي  | المنطقة الصناعية، الجزائر     |
| منطقة    | A1             | الممر 1           | نوع AISLE                     |
| أماكن    | A1-01 إلى A1-05 | رف 1 إلى 5       | سعة قصوى: 100 وحدة           |

## 9. البحث الشامل

### الوصول إلى البحث

- انقر على **شريط البحث** في شريط التنقل العلوي
- أو استخدم اختصار لوحة المفاتيح **Ctrl + K**

### الاستخدام

1. اكتب حرفين على الأقل في شريط البحث
2. تظهر النتائج فوراً (بحث في الوقت الحقيقي مع تأخير 300 مللي ثانية)
3. يتم تجميع النتائج حسب الفئة:

| الفئة            | الأيقونة | الحقول المبحوث فيها                      |
|-----------------|---------|------------------------------------------|
| **المنتجات**     | 📦      | الاسم، SKU، الرمز الشريطي               |
| **الفئات**       | 🗂️      | الاسم + عدد المنتجات المرتبطة           |
| **الحركات**      | ↔️      | المرجع، النوع، المنتج المرتبط           |
| **الأماكن**      | 📍      | التسمية، الرمز، المنطقة والموقع الأب    |

### البحث الصوتي 🎤

إذا كان متصفحك يدعم التعرف على الصوت (Web Speech API):

1. يظهر زر **ميكروفون** (🎤) على يمين شريط البحث
2. انقر عليه لتفعيل الاستماع (ينبض الزر باللون الأحمر)
3. تحدث بوضوح — يتم إدخال النص المُتعرَّف عليه تلقائياً في شريط البحث
4. انقر مجدداً لإيقاف الاستماع

> متوافق مع Chrome و Edge والمتصفحات المبنية على Chromium.

### التنقل في النتائج

- استخدم مفتاحي السهم **↑** و **↓** للتنقل
- اضغط **Enter** لفتح العنصر المحدد
- اضغط **Escape** لإغلاق البحث

## 10. الإعدادات

متاحة عبر القائمة الجانبية **الإعدادات**. تحتوي الصفحة على 5 لوحات.

### المظهر (العلامة البيضاء)

خصّص مظهر التطبيق:

| الخيار               | الوصف                                          |
|---------------------|-------------------------------------------------|
| **اللون الرئيسي**    | 10 ألوان محددة مسبقاً + شريط تمرير مخصص        |
| **وضع الشريط الجانبي** | فاتح أو داكن                                 |
| **اسم التطبيق**     | استبدال «خزانة-DZ» بعلامتك التجارية            |
| **الشعار**          | رفع ملف صورة (PNG، JPG، SVG أو WebP — أقصى 2 ميغابايت) |

تُطبّق التغييرات فوراً وتُحفظ في المتصفح (localStorage).

### اللغة

اختر لغة الواجهة:

| اللغة     | الرمز | الاتجاه |
|----------|-------|---------|
| Français  | FR    | LTR     |
| English   | EN    | LTR     |
| العربية   | AR    | RTL     |

اختيار العربية يحوّل الواجهة تلقائياً إلى وضع **من اليمين إلى اليسار** (RTL).

### إدارة المستخدمين

زر **إدارة المستخدمين** يعيد التوجيه إلى الصفحة المخصصة `/users` (انظر [القسم 11 — إدارة المستخدمين](#11-إدارة-المستخدمين)).

### الحقول المخصصة

أضف حقولاً إضافية إلى بطاقات المنتجات:

1. انقر على **+ إضافة حقل**
2. املأ:
   - **الاسم** (مثال: «الوزن الصافي»، «المورّد»)
   - **النوع**: TEXT (افتراضياً)
   - **مطلوب**: نعم/لا
3. انقر على **إنشاء**

لحذف حقل، انقر على أيقونة **سلة المحذوفات** بجانب الحقل.

### عتبات المخزون

حدد عتبات التنبيه لتلقي إشعارات عند انخفاض المخزون:

1. انقر على **+ إضافة عتبة**
2. اختر **المنتج**
3. حدد العتبات الثلاث:
   - **المخزون الأدنى** — الكمية التي يُفعَّل دونها التنبيه
   - **مخزون الأمان** — هامش أمان إضافي
   - **نقطة الطلب** — الكمية التي يُنصح عندها بإعادة التموين
4. انقر على **إنشاء**

عندما ينخفض مخزون منتج دون العتبة، يتم إطلاق تنبيه تلقائياً (مرئي في صفحة **التنبيهات**).

### الفئات

أدِر شجرة فئات المنتجات:

1. انقر على **+ إضافة فئة**
2. أدخل **اسم** الفئة
3. اختر اختيارياً **فئة أصل**
4. انقر على **إنشاء**

تدعم الفئات تداخلاً غير محدود (فئات وفئات فرعية).

## 11. إدارة المستخدمين

متاحة عبر القائمة الجانبية **المستخدمون** (مرئية فقط لأدوار ADMIN و MANAGER).

### عرض المستخدمين

تعرض الصفحة قائمة بجميع المستخدمين مع:
- **الصورة الرمزية** (الأحرف الأولى ملونة)
- **الاسم الكامل** وعنوان البريد الإلكتروني
- **الدور** (شارة ملونة: أحمر ADMIN، أزرق MANAGER، أخضر OPERATOR، رمادي VIEWER)
- **الحالة** (نشط / غير نشط)
- **تاريخ الإنشاء**

### إنشاء مستخدم (ADMIN فقط)

1. انقر على زر **+ مستخدم جديد**
2. املأ النموذج:

| الحقل              | الوصف                          | مطلوب |
|------------------|--------------------------------|:-----:|
| **الاسم الكامل**    | الاسم واللقب                    | ✅    |
| **البريد الإلكتروني** | عنوان بريد إلكتروني فريد  | ✅    |
| **كلمة المرور**    | 6 أحرف كحد أدنى                   | ✅    |
| **الدور**          | ADMIN / MANAGER / OPERATOR / VIEWER | ✅ |

3. انقر على **إنشاء**
4. يتم تشفير كلمة المرور تلقائياً (bcrypt) قبل التخزين

### تعديل مستخدم (ADMIN فقط)

1. انقر على أيقونة **القلم** في سطر المستخدم
2. عدّل الحقول المطلوبة (الاسم، البريد الإلكتروني، الدور)
3. حقل كلمة المرور **اختياري** — اتركه فارغاً للإبقاء على كلمة المرور الحالية
4. انقر على **حفظ**

### تفعيل / تعطيل مستخدم (ADMIN فقط)

- انقر على زر **التبديل** في عمود الحالة
- المستخدم المعطّل لا يمكنه تسجيل الدخول (يتم التحقق بواسطة استراتيجية JWT)

### حذف مستخدم (ADMIN فقط)

- انقر على أيقونة **سلة المحذوفات** في سطر المستخدم
- أكد الحذف
- المسؤول المتصل لا يمكنه حذف نفسه

### وصف الأدوار

في أسفل الصفحة، 4 بطاقات تصف مسؤوليات كل دور:

| الدور        | اللون   | النطاق                                          |
|-------------|--------|----------------------------------------------------|
| **ADMIN**    | أحمر  | وصول كامل، إدارة المستخدمين، التدقيق، الحذف     |
| **MANAGER**  | أزرق  | إدارة المنتجات، الحركات، الموردين، العملاء   |
| **OPERATOR** | أخضر  | إنشاء حركات المخزون فقط                       |
| **VIEWER**   | رمادي | للقراءة فقط                                      |

## 12. الأدوار والصلاحيات

يستخدم التطبيق نظام التحكم في الوصول المبني على الأدوار (RBAC) بأربعة مستويات.

### نظرة عامة

| الدور        | الوصف                                        | الوصول                                   |
|-------------|----------------------------------------------|------------------------------------------|
| **ADMIN**    | مسؤول كامل الصلاحيات                        | وصول كامل، إعدادات، مستخدمين، تدقيق |
| **MANAGER**  | مسؤول الموقع                                | إدارة المنتجات، الحركات، التنبيهات، الموردين، العملاء |
| **OPERATOR** | مشغّل المخزون                                | إنشاء الحركات فقط                   |
| **VIEWER**   | للقراءة فقط                                  | وصول للقراءة فقط لجميع الصفحات       |

### مصفوفة الصلاحيات المفصلة حسب الوحدة

| الوحدة              | العرض              | الإنشاء            | التعديل            | الحذف            |
|---------------------|-------------------|-------------------|-------------------|-------------------|
| **المنتجات**        | كل مصادَق عليه    | ADMIN, MANAGER    | ADMIN, MANAGER    | ADMIN             |
| **الفئات**          | كل مصادَق عليه    | ADMIN, MANAGER    | ADMIN, MANAGER    | ADMIN             |
| **الحركات**         | كل مصادَق عليه    | ADMIN, MANAGER, OPERATOR | —          | —                 |
| **التخزين (المواقع)**| كل مصادَق عليه | ADMIN, MANAGER    | ADMIN, MANAGER    | ADMIN             |
| **التنبيهات**       | كل مصادَق عليه    | —                 | ADMIN, MANAGER    | —                 |
| **العتبات**          | كل مصادَق عليه    | ADMIN, MANAGER    | ADMIN, MANAGER    | ADMIN, MANAGER    |
| **الموردون**        | كل مصادَق عليه    | ADMIN, MANAGER    | ADMIN, MANAGER    | ADMIN             |
| **العملاء**          | كل مصادَق عليه    | ADMIN, MANAGER    | ADMIN, MANAGER    | ADMIN             |
| **المستخدمون**      | ADMIN, MANAGER    | ADMIN             | ADMIN             | ADMIN             |
| **الحقول المخصصة**  | كل مصادَق عليه    | ADMIN             | ADMIN             | ADMIN             |
| **التدقيق (السجلات)**| ADMIN          | —                 | —                 | —                 |
| **الاستيراد/التصدير** | —              | ADMIN, MANAGER    | —                 | —                 |
| **البحث**            | كل مصادَق عليه    | —                 | —                 | —                 |
| **مستويات المخزون**| كل مصادَق عليه | —                 | —                 | —                 |

> **كل مصادَق عليه** = ADMIN + MANAGER + OPERATOR + VIEWER (أي مستخدم متصل).

يتم تعيين المستخدمين إلى موقع تخزين واحد أو أكثر، مما يحد من نطاق عملهم.
